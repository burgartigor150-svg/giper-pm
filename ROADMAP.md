# ROADMAP.md

## Принцип

Каждая фаза = working software. После завершения фазы система **уже годна
к использованию**, даже если функционала меньше. Не строим месяцами «вглубь»
без видимого результата.

В конце каждой фазы — короткий self-review: что работает, что отложено,
что узнали по ходу. Обновляется этот файл.

---

## Фаза 1: MVP веба (1–2 недели)

**Цель**: команда может создавать задачи, двигать по канбану, логировать время вручную.

### Что входит
- [x] Скаффолд монорепо (pnpm + Turborepo)
- [ ] Next.js 15 + Tailwind + shadcn/ui
- [ ] Prisma + Postgres локально (Docker Compose)
- [ ] NextAuth: Google OAuth + Email magic-link
- [ ] RBAC middleware (admin/pm/member/viewer)
- [ ] Crud проектов + добавление участников
- [ ] Crud задач + статусы + assignee + estimate
- [ ] Канбан-доска (drag-n-drop, dnd-kit)
- [ ] Список задач с фильтрами и сортировкой
- [ ] Детальная страница задачи + комментарии
- [ ] Live timer (Start/Stop) на задаче
- [ ] Ручной ввод времени (форма)
- [ ] «Мой день» — личный дашборд (текущие задачи, таймер, часы за сегодня)
- [ ] «Команда» — список активных таймеров команды (для PM)
- [ ] Базовый отчёт «Время по проектам за период» (CSV экспорт)
- [ ] Аудит-лог изменений задач

### Что НЕ входит (откладываем)
- Gantt, sub-tasks > 1 уровня
- Любые автотрекеры, агенты, расширения
- Telegram-бот
- Внешние интеграции
- Скриншоты
- i18n (только русский)

### Done criteria
- 5 человек могут параллельно работать
- За день работы все часы команды видны PM-у в одной таблице
- Никаких ошибок 500 при базовых сценариях
- Lighthouse Performance > 80 на главных экранах

## Фаза 2: Telegram-бот + GitHub-интеграция (3–5 дней)

**Цель**: мобильный ввод времени + автокомментарии в задачи из коммитов.

### Что входит
- [ ] Telegram-бот на grammY
- [ ] Команды: /start, /stop, /today, /week, /log, /pause
- [ ] Pairing TG-чата с юзером (через код в вебе)
- [ ] Inline-клавиатура для выбора задачи
- [ ] Ежедневный пуш в 18:00 «подтверди часы»
- [ ] GitHub webhook handler: push, pull_request, pull_request_review
- [ ] Парсинг номеров задач из commit messages (`GFM-42` → задача GFM-42)
- [ ] Авто-комментарий в задачу со ссылкой на коммит/PR
- [ ] Auto-status: PR opened → задача в Review, PR merged → задача в Done
  (опционально, выключаемо в настройках проекта)

### Done criteria
- Я могу залогировать час с телефона за 10 секунд
- Все коммиты команды автоматически линкуются к задачам
- PM получает в TG ежедневный дайджест команды

## Фаза 3: Desktop-агент (1 неделя)

**Цель**: автоматический трекинг активности с явными согласиями.

### Что входит
- [ ] Electron-проект с auto-update
- [ ] Pairing с вебом через код
- [ ] active-win каждые 30 сек
- [ ] node-system-idle-time, авто-пауза при idle > 5 мин
- [ ] Tray-меню: текущая задача, Start/Stop, «Я не на работе»
- [ ] Экран согласий при первом запуске (4 чекбокса)
- [ ] Экран «Мои данные» — что собирается, превью последних N часов
- [ ] Endpoint /api/agent/activity (батч-загрузка раз в 5 мин)
- [ ] Endpoint /api/agent/heartbeat (раз в минуту, для статуса онлайн)
- [ ] Категоризация по правилам (rules в JSON, см. INTEGRATIONS.md)
- [ ] Маппинг окно → задача (по ключевым словам в title, по проекту в IDE)
- [ ] Билд .exe / .dmg / .AppImage через electron-builder

### Откладываем на 3.5
- Скриншоты (отдельно из-за этической чувствительности)

### Done criteria
- Агент работает 8ч непрерывно без падений
- Соответствие manual ↔ auto > 80% на тесте «программист день кодит»
- Расход RAM агентом < 200 МБ

## Фаза 3.5: Скриншоты (опционально, 2 дня)

- [ ] Опт-ин чекбокс в согласиях (default OFF)
- [ ] Захват каждые 10 мин, downscale 800px, PNG ~50КБ
- [ ] Локальное превью в трее с кнопкой «удалить последний»
- [ ] Опциональный blur (heuristic: всё кроме окна активного приложения)
- [ ] Загрузка в S3 (Cloudflare R2)
- [ ] Cron auto-delete после 30 дней
- [ ] Просмотр своих скринов на /me/data

### Done criteria
- Сотрудник может в любой момент удалить любой свой скриншот
- Cron реально удаляет старые скрины (не просто помечает)
- Никто кроме сотрудника и его прямого руководителя не видит скрины
